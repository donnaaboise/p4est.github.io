Create and manipulate a forest of octrees

p4est represents a mesh topology as a forest of octrees. This forest can be created by p4est given a connectivity. In this tutorial, we talk about the typical workflow in p4est to create and manipulate a forest of quadtrees (2D) or octrees (3D) to represent a mesh topology.

Build, Connectivity and optional VTK graphics output in IO
Skills acquired
Create a forest of quadtrees and change the refinement structure of the created forest.

Create a forest of octrees

The first step of the typical workflow of p4est is to create a forest of octrees or quadtrees. The p4est libary offers the function p4est_new and p4est_new_ext (a version with more parameters) to create such a forest. 

typedef my_user_data {
  int foo;
} my_user_data_t;
static void my_quadrant_init (p4est, which_tree, quadrant) {
  ((my_user_data_t *) quadrant->p.user_data)->foo = 17948;
}
void *user_pointer = NULL;
p4est = p4est_new_ext (mpicomm, connectivity, 0, level, 1,
                       sizeof (my_user_data_t), my_quadrant_init, user_pointer);

mpicomm is an MPI communicator (see the example simple for a usage example). The minimum level of refinement is specified. If it is zero or negative, there will be no refinement beyond the coarse mesh connectivity at this point. We provide an optional callback mechanism to initialize the user data that we allocate inside any forest leaf. The user_pointer is assigned to the member of the same name in the p4est created (before the init callback function is called the first time). This is a way to keep application state referenced in a p4est object. For more details see p4est.h and p4est_extended.h (as always, the 3D equivalents are prefixed with p8est_).

Exercise F1: Choose in p4est_connectivity.h a connectivity that you like and create a forest of quadtrees.

Manipulate the refinement structure and the partition

The hypercubes for the dimensions two and three are called quadrants (for three dimensions also octants). In p4est.h the struct p4est_quadrant_t is defined and documented.

Our next step in the workflow is to manipulate the refinement structure. The p4est libary offers the following functions to manipulate the forest of quadtrees (2D) or octrees (3D).

    p4est_refine
    Refinement of specific hypercubes given a refinement criterion, i.e a callback function.
    p4est_coarsen
    Coarsen a family of hypercubes given a coarsening criterion, i.e a callback funtion.
    p4est_balance
    This function ensures a 2:1 balance of the size differences of neigboring elements in a forest by refining quadrants.
    p4est_partition
    Partition the forest equally or according to a given user-defined weight.

For the declaration and documentation of the above functions you can use the links in the box above.

Exercise F2: Use p4est_refine to refine the boundary (choose a boundary thickness) of a circle until a the quadrants reached the refinement level six and redistribute the quadrants between the processes using p4est_partition.

Exercise F3: Coarsen all families of quadrants that have a level greater than four and call p4est_balance on the forest.

Exercise F4: Use the function p4est_vtk_write_file to create a VTK file for the created forest.
